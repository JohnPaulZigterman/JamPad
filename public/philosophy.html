JamPad is a compact generative trip-hop instrument: beginner-friendly on the surface, disproportionately powerful underneath, and opinionated enough to make musical choices without becoming a general-purpose DAW.

North Star

Build a focused probability instrument, not a miniature Ableton. The machine should breathe between silence and material, know the difference between roles, remember recent choices, favor the curated crate, and let a beginner keep good accidents without understanding the whole system. When tradeoffs happen, the playable hardware instrument wins by a little. The feeling should sit between playing an instrument and playing a slightly cryptic game that does not quite want to tell you all of its rules.

Quiz Results

Current product taste from the design quiz. JamPad should invite play first, preserve vibe by default, and reveal deeper intelligence only when the user reaches for it. It should reward rule-hunting without requiring rule mastery.

Identity Priority

Hardware First, Lightly

JamPad should feel most like a playable hardware instrument, with composition-partner and sample-lab powers supporting that identity instead of overtaking it.

Automation Taste

Preserve The Vibe

The engine should mostly protect the current mood, then add slightly unstable interruptions: stingers, fills, dropouts, phrase mutations, and small collapses.

Surface Power

Macros On The Faceplate

Fundamental controls and hyper-useful macros belong on the main surface. Advanced options can live under quiet menus, but they should still be quick to reach.

Macro Layers

Expressive Plus Literal

Use both expressive macros and literal controls. A Bloom/Break/Recover layer can interact with Density/Silence/Weirdness/Stability rather than replacing them.

Samples Language

Ecology In Musical Terms

Conway-inspired behavior should use musical names: stingers, vamps, riffs, fills, anchors, dropouts, bridges, swells, ghosts, and ruptures.

Learning Discovery

Teach When Invited

Discovery stays in front. Tutorials should be easy to find, useful, and optional, never a gate before the user can play.

Mood Game Logic

Slightly Cryptic Game

The machine should feel like it has hidden rules, rewards, and strange little tells, but not like a puzzle that blocks music. Mystery should pull the user deeper into play.

Experience Map

Keep this map visible in planning. JamPad can support several user experiences, but they should all orbit one playable loop: start sound quickly, steer the machine, keep good accidents, and only open deeper tools when the user asks for them.

Entry Open JamPad

Curated crate loads first. The user sees a compact instrument, not a setup chore.

First Gesture Press Play

The machine starts making decisions with safe defaults, calmer phrase holds, tempo snapping, and key lock.

Steer Mode + Macros

Sparse, Groove, Collapse, density, silence, weirdness, hold, tempo, and key shape taste.

Intervene Lock / Mute / Next

The user keeps a good lane, clears space, or asks the machine for a new decision.

Commit Keep / Export / Build

Save the accident, render a performance, or open the crate workshop when ready.

Beginner Path

Play First

Open, press Play, change Mode, hit Next. The workflow must stay usable with no crate editing and no theory knowledge.

Performer Path

Shape And Capture

Use locks, mutes, key, tempo, and macros to shepherd a living performance, then keep the moment.

Builder Path

Curate The Machine

Import or buy samples, tag roles, check keys and BPM, then return to the same compact instrument surface.

Three Tech Trees

These are the main development lenses. Every new idea can be placed under one tree, blended across two, or treated as a mutation of all three. Each tree keeps beginner friendliness, hyper usability, and compact power in view.

01. Pocket Instrument

Make JamPad feel like a small piece of hardware that a new user can play in ten seconds, then gradually discover deeper behavior.

  • Surface Play, Next, Mode, Lock, Mute, Ctrl. Nothing decorative competes.
  • Mutation Beginner Mode, Performance Mode, and Crate Edit Mode.
  • Payoff The app looks simple, but every button has musical weight.

02. Taste Engine

Improve the machine's musical judgment: key awareness, density arcs, mood memory, role compatibility, vibe preservation, and controlled misbehavior.

  • Surface Users steer through expressive macros, with literal controls tucked nearby.
  • Mutation Key lanes, phrase memory, rare-event grammar, scene moods, and small unstable interruptions.
  • Payoff The output feels directed rather than shuffled.

03. Crate Workshop

Make sample selection, tagging, key matching, and role assignment fast enough that users can build their own machine without becoming librarians.

  • Surface Curated default crate, then import and swap when ready.
  • Mutation Loopcloud-style search notes, audition stacks, smart tagging, and key locks.
  • Payoff Personalization feels like browsing a record bin, not maintaining a database.

Direction Catalogue

Browse these like catalogue spreads. None are commitments by themselves; they are proposals to pull into the roadmap when they serve the instrument.

Pocket Instrument Beginner First

One-Screen Performance Mode

A locked-down view for playing: no clip lists, no file concerns, only mode, lanes, locks, mutes, macro controls, and a small workflow map showing where the user is.

  • Great for first-time use and live capture.
  • Pairs well with keyboard shortcuts and MIDI learn.
  • Risk: hiding too much unless Ctrl and the current path are obvious.
Taste Engine Key Aware

Home Key And Compatibility

Let the machine know a home key, then prefer compatible bass, chord, texture, and vocal material while leaving drums and noise mostly free. Current build: a compact Key control, Key Lock, and weighted harmonic selection.

  • Beginner surface: one home key and one lock switch.
  • Advanced layer: later show actual pitch offsets and suggested transposes.
  • Mutation: temporary key drift during Collapse.
Taste Engine Timing

Tempo-Smart Crate

Compare every sample to the session tempo using half-time, normal-time, and double-time interpretations, then stretch looped playback toward the closest musical fit.

  • Beginner result: loops just sit in the pocket.
  • Current build: looped samples snap to global BPM with clean granular stretching.
  • Mutation: allow wild rates during Collapse for warped tape moments.
Taste Engine Phrasing Built

Breathing Loop Memory

Individual clips and silent rests should tend to stay active for another pass before the machine asks for a new event. Probability still leads, but phrases get room.

  • Beginner result: fewer frantic switches.
  • Current build: stronger clip holds, rest holds, per-lane stickiness, and lower return-to-silence churn.
  • Mutation: Collapse can still cut faster when the user asks for instability.
Pocket Instrument Built

Lane Behavior Trims

Each lane gets two tiny faceplate trims: RND for exploratory choice and STK for looping/holding behavior. They keep the instrument playable without opening a rules editor.

  • Beginner result: drums can stay sticky while fills stay unstable.
  • Cryptic-game result: every lane feels like it has a temperament.
Taste Engine Mixing

Automatic Crate Leveling

Measure each decoded sample, trim it toward a role-aware loudness target, and protect against peaks so the user does not have to babysit every purchased loop.

  • Beginner result: Play First stays comfortable by default.
  • Advanced surface: a single Auto Mix switch, not eight tiny faders.
Pocket Instrument Workflow

Always-Visible Path Map

Add a compact visual breadcrumb to the app and keep the larger Field Guide map as the planning source of truth. The user should always know whether they are playing, steering, capturing, or building.

  • Beginner payoff: fewer mystery modes.
  • Product payoff: roadmap discussions stay grounded in real journeys.
Crate Workshop

Import Wizard

Drop in a folder and let JamPad propose roles, BPM, density, weirdness, and key from file names and audio analysis.

  • Accept all, review uncertain, or audition by role.
  • Keep the default crate as the fallback voice.
Crate Workshop Built

Pruned Loopcloud Batch

Imported a larger Loopcloud batch, then pruned the active starter crate to 66 clips by removing harsh, novelty-coded, high-energy, and off-theme material.

  • Pitch-locked exports use uncertain key hints, shown as a question-mark key in the app.
  • Selection can still benefit from filename keys without treating them as gospel.
  • Removed files stay on disk for recovery, but no longer clutter the default instrument.
Taste Engine

Scene Moods

Smoky, Dusty, Underwater, Tense, Pretty, Broken. Moods bias roles and clips without becoming arrangement timelines.

  • Beginner control: one mood knob or menu.
  • Mutation: Auto drifts every 16 or 32 bars.
Pocket Instrument

Hardware Skin Set

Alternate compact visual skins inspired by sampler eras: black/yellow utility, red/grey performance, cream/blue studio, and night mode.

  • All skins preserve the same control layout.
  • Style should never cost readability.
Taste Engine Usability

Good Accident Capture

When a pattern feels right, one button freezes the current combination as a snapshot or exports a short performance.

  • Beginner text: `Keep This`.
  • Advanced option: save snapshot with settings and active clip IDs.
Crate Workshop Loopcloud

Crate Shopping Notes

Maintain a living acquisition log: useful searches, point spend, roles bought, harmonic cluster, and what still feels missing.

  • Current search lesson: short terms plus table metadata beat verbose searches.
  • Keep key matching visible for melodic purchases.
Architecture Efficiency

Lazy Audio Loading

Load sample players only when a clip is likely to play, cache recent sounds, and release unused buffers so the app stays light as crates grow.

  • Protects startup speed.
  • Pairs with role-based preloading.
Architecture Timing

Audio Worklet Upgrade

Keep Tone.js while the product is young, but reserve a path for tighter scheduling, metering, and recording if timing pressure grows.

  • Do this only when current scheduling becomes a real blocker.
  • Avoid premature engine rewrites.
Hybrid Mutation Pocket Instrument + Taste Engine + Crate Workshop

The Disproportionately Powerful Box

The strongest version of JamPad looks modest: eight lanes, a handful of knobs, and one curated crate. Underneath, it has role memory, key-safe selection, scene moods, density arcs, lockable accidents, and smart crate import. The beginner gets music immediately; the expert gets a compact probability instrument with taste.

Idea Market

Shop here when building a long-term plan. These ideas are designed as bundles: small beginner-facing controls on top of automation, macro behavior, and power-user depth.

Automation Macro Built

Director Macro

One big performance control that turns several taste settings together: density, silence, weirdness, stability, fill probability, and scene drift.

  • Positions: Preserve, Nudge, Bloom, Stinger, Break, Recover.
  • Current build: the faceplate macro now biases the probability engine.
  • Beginner payoff: steer a whole performance without learning every parameter.
  • Power layer: users can edit the literal controls each macro moves later.
Automation Taste Engine

Auto Scene Drift

Let the machine move between Sparse, Groove, and Collapse over 8, 16, or 32-bar phrases while keeping the current mode visible and interruptible.

  • Modes: Off, Gentle, Restless, Livewire.
  • Pairs with Conway-style stable forms and graceful extinction.
Automation Capture

Good Take Detector

Watch for moments with strong balance, low clipping risk, useful density, and recent novelty, then light a compact Keep button.

  • Beginner surface: Keep This.
  • Advanced surface: save reason tags like tight, rare, spacious, or collapse.
Macro Arrangement

Energy Arc Presets

Preset curves for how the machine should behave across time without becoming a fixed song arrangement.

  • Shapes: Slow Burn, Smoke Rings, False Drop, Tape Collapse, Late Night Loop.
  • Every arc should still allow locks, mutes, and manual Next gestures.
Beginner Compactness

Three-Layer Interface

Condense the app into Play, Tune, and Build layers. Play gets the instrument, Tune gets macro settings, Build gets crate and sample management.

  • Default layer: Play.
  • Rule: no feature appears in Play unless it helps live performance.
  • Power stays available through deliberate layer changes.
Beginner Usability Built

Confidence Rail

A tiny status strip that explains what the machine is doing in plain musical terms: holding groove, opening space, inviting bass, cooling down, or searching.

  • Current build: Hidden Rule tells expose just enough of the machine's logic.
  • Keeps mystery low without writing tutorials into the performance surface.
  • Can double as a workflow breadcrumb.
Beginner Controls

Smart Defaults And Guardrails

Beginner-safe ranges for tempo, weirdness, density, and silence, with an Advanced switch for wider extremes.

  • First-run mode should make it hard to create ugly chaos accidentally.
  • Advanced mode can expose unstable behavior intentionally.
Crate Workshop Sorting Built

Sample Drawers

Store samples in drawers by role, density, key confidence, tempo family, mood, and ecological-musical trait. Drawers should feel like record-bin browsing, not spreadsheets.

  • Role drawers: drums, bass, chords, texture, vocal, percussion, fills, noise.
  • Feel drawers: stinger, vamp, riff, fill, anchor, dropout, bridge, swell, ghost, rupture.
  • Current build: clips carry musical ecology tags used during selection.
  • Uncertain metadata gets a visible but non-punitive question mark.
Crate Workshop Storage

Crates, Stacks, And Snapshots

Separate permanent sample crates, temporary audition stacks, and saved performance snapshots so users do not confuse shopping, experimenting, and preserving.

  • Crate: long-term library.
  • Stack: short audition pile for tonight.
  • Snapshot: exact machine state worth returning to.
Crate Workshop Automation

Auto Tagging Pass

Analyze imported samples for approximate tempo, duration, loudness, transient feel, likely role, and filename key hints, then let the user approve the uncertain bits.

  • Beginner result: imports are playable quickly.
  • Power result: metadata improves selection without manual library chores.
Crate Workshop Built Internet Radio

Play The Internet

Let JamPad tune public audio sources like a strange radio: search, preview, analyze, and propose public-domain or Creative Commons fragments that can become local crate clips after user approval.

  • Current build: manual files and Internet Archive offerings can be staged, auditioned, accepted, or banished.
  • Beginner surface: Source, Mood, Weirdness, Density, Harvest.
  • Hidden rule feel: the machine finds offerings, omens, ghosts, stingers, and vamps.
  • Trust rule: source, license, creator/title, and original URL stay attached.
  • Safety rule: no silent import of unknown copyrighted material.
Automation Internet Built

Harvest Ritual

A compact flow for internet sampling: tune a source, audition machine-picked candidates, accept or banish, then let Auto Tagging assign role, BPM, key confidence, ecology, and mix level.

  • Current build: Internet Archive search uses fast metadata and filename heuristics first.
  • Feels like playing a dial, not managing downloads.
  • Rejected sounds teach the machine what not to bring back.
Architecture Rights

Source Ledger

Every internet-derived clip keeps provenance: source platform, item URL, creator, title, license URL, retrieval date, transform notes, and whether the user approved it for the crate.

  • Required for trust, export, and later cleanup.
  • The game can be cryptic; the rights trail cannot.
Macro Mixing

One-Knob Mix Posture

A compact mix macro that shifts role balance without eight faders: Drums Forward, Bass Weight, Ghost Room, Vocal Dust, or Flat Utility.

  • Pairs with automatic crate leveling.
  • Beginner surface: one selector, not a mixer page.
Automation Performance

Scheduled Interventions

Let users queue simple future actions: mute fills for 4 bars, open texture next phrase, force rest after Collapse, or bring drums back on the next downbeat.

  • Feels like conducting, not editing a timeline.
  • Good bridge between generative play and arrangement control.

Tutorial Beginner

First Beat Tour

A two-minute guided mode where the user presses Play, locks a lane, changes mode, hits Next, and saves a snapshot.

  • Never blocks the instrument after first run.
  • Can be replayed from Help as a practice routine.
Tutorial Learning

Explain The Machine

Optional overlays that explain one behavior at a time: why a lane held, why silence persisted, why a sample matched key, or why Collapse broke the pattern.

  • Teaches through current behavior, not abstract docs.
  • Could reuse the Confidence Rail language.
Tutorial Mystery

Rule-Hunting Mode

A light optional mode that hints at hidden machine behavior without spelling out the whole system: notice what happens after a long rest, a locked anchor, or a Collapse.

  • Best for: making discovery feel intentional.
  • Rule: hints should invite experiments, not lecture.
Tutorial Power Path

Technique Cards

Short, optional cards for useful moves: make space, build pressure, freeze an anchor, recover from chaos, shop a crate, or tune key confidence.

  • Feels like learning an instrument, not onboarding software.
  • Can become a long-term practice and documentation system.

Conway Lens

Conway's Game of Life is not a theme for the UI. It is a discipline for the machine: tiny local rules should create musical behavior that feels bigger than the controls.

Taste Engine Emergence

Local Rules, Global Groove

In Life, a cell lives or dies based on its neighbors. In JamPad, a lane should eventually choose, hold, rest, or mutate based on nearby musical context: drums can encourage bass, bass can invite chords, dense lanes can starve fills, and long rests can make the next entrance feel earned.

  • Beginner result: the machine feels alive without exposing a rules editor.
  • Product rule: neighbor influence should be audible as taste, not visible as math.
  • Implementation path: start with role adjacency weights before adding spatial grids.
Taste Engine Patterns

Still Lifes And Oscillators

Some combinations should settle into stable forms, while others should pulse between two or three related states. This gives JamPad recognizable behavior without fixed arrangements.

  • Still life: drums, bass, and texture hold until the user nudges them.
  • Oscillator: vocal dust and percussion trade places every few bars.
  • Collapse mutation: stable forms can intentionally overcrowd and break.
Pocket Instrument Usability

Readable Emergence

Life is fascinating because the rules are simple enough to trust. JamPad should keep this same readability: the user changes Mode, Stability, Silence, and Weirdness, then hears the ecosystem respond.

  • Do not add a visible cellular automata simulator unless it improves performance.
  • Use compact indicators: stable, pulsing, crowded, recovering.
  • Keep Play First intact; emergence should be felt before it is studied.
Crate Workshop Ecology

Musical Clip Ecology

Samples can carry ecology through musician-friendly roles: anchor, vamp, riff, stinger, fill, dropout, bridge, swell, ghost, rupture. These tags let imported crates behave musically without making every clip equally likely.

  • Vamps and anchors stabilize a section.
  • Stingers and fills create slightly unstable interruptions.
  • Dropouts and bridges clear density after overcrowded moments.

Decision Shelf

Use this shelf when comparing directions. The bars score how strongly each proposal supports beginner friendliness, compactness, and musical consequence.

Prioritize: sample crate on load

Default to the curated crate because it makes the product sound intentional before the user has customized anything.

Built: key-aware and tempo-aware selection

The crate now prefers compatible keyed material and nearby half/normal/double-time BPM matches, with a compact Key Lock surface and global tempo-snapped loop playback.

Built: calmer phrase behavior

Clips and silences now hold more often, with per-lane STK trims so JamPad breathes in short musical phrases instead of constantly re-rolling every bar.

Built: no accidental dead air

If any unmuted lane is available, the machine now forces at least one sounding clip instead of drifting into total extended silence.

Inform: Conway-style emergence

Future Taste Engine work should prefer simple neighbor rules that create stable forms, oscillations, blooms, and graceful collapse without adding beginner friction.

Prioritize: faceplate macros

Director macros now live on the surface. Next: let unobtrusive literal controls shape each macro without crowding the front panel.

Prioritize: musical ecology tags

Clips now carry musical ecology tags like stinger, vamp, riff, fill, anchor, dropout, bridge, swell, ghost, and rupture, and those tags influence machine choice.

Inform: cryptic game feel

Let JamPad hide some rules and rewards under the surface, as long as the first experience still feels playable, musical, and beginner-safe.

Built: automatic crate leveling

Samples now get role-aware gain staging and peak-safe normalization so volume cleanup happens under the hood.

Next likely build: workflow breadcrumb

Add a tiny in-app path indicator so users can see Play, Steer, Capture, and Build options without opening the full Field Guide.

Later: user crate import

Important, but it should arrive after the default instrument feels delightful. The workshop should not bury the first-time player.

Built: first internet radio

Manual import and Internet Archive harvest now stage candidates for user approval. Next: better license filters, local caching, and deeper post-accept analysis.

Built: pruned starter crate

The crate now favors a tighter 66-clip voice, keeping pitch-locked filename keys as useful but uncertain metadata while removing clutter from the active starter list.

Defer: DAW-scale editing

Arrangers, piano rolls, plugin chains, and deep project management are outside the current instrument promise.
